# [OpenDART] 단일회사 주요계정 API 개발 가이드

이 문서는 OpenDART에서 제공하는 **단일회사 주요계정** API에 대한 상세 개발 가이드입니다. 이 API는 특정 기업의 재무상태표 및 손익계산서에서 대차대조표의 주요 항목(자산, 부채, 자본, 매출, 이익 등)을 빠르게 조회할 때 사용됩니다.

## 1. 기본 정보

| 항목 | 상세 내용 |
| :--- | :--- |
| **메서드** | GET |
| **요청 URL** | `https://opendart.fss.or.kr/api/fnlttSinglAcnt.json` (또는 `.xml`) |
| **인코딩** | UTF-8 |
| **출력 포맷** | JSON, XML |

---

## 2. 요청 인자 (Parameters)

| 요청키 | 명칭 | 타입 | 필수여부 | 값 설명 |
| :--- | :--- | :--- | :---: | :--- |
| `crtfc_key` | API 인증키 | STRING(40) | Y | 사용자가 발급받은 40자리의 인증키 |
| `corp_code` | 고유번호 | STRING(8) | Y | 공시대상회사의 고유번호 (8자리)<br>※ 고유번호 API 참고 |
| `bsns_year` | 사업연도 | STRING(4) | Y | 사업연도 (4자리, 2015년 이후 정보 제공) |
| `reprt_code` | 보고서 코드 | STRING(5) | Y | 아래 코드값 중 선택<br>1분기: `11013`, 반기: `11012`, 3분기: `11014`, 사업보고서: `11011` |

---

## 4. 메시지 설명 (Status Codes)

주요 코드만 발췌하였습니다. 자세한 내용은 `api_corp_code.md` 문서를 참고하십시오.
- `000`: 정상
- `013`: 조회된 데이터가 없음 (데이터 가공 중이거나 해당 연도 미공시)

---

## 5. 개발 참고 사항 및 활용 팁

### 5.1 데이터의 성격
이 API는 재무제표 전문(Full Text)을 반환하는 것이 아니라, 투자자가 가장 많이 찾아보는 **핵심 계정 과목**만 선별하여 제공합니다. 따라서 상세 주석이나 모든 계정 과목이 필요한 경우에는 `재무제표(XBRL) API`나 `전체 재무제표 API`를 사용해야 합니다.

### 5.2 조회 가능한 회사와 연도
2015년 이후 제출된 보고서부터 정보가 제공됩니다. 또한, 코스닥/코스피 등 상장회사 위주로 표준화된 데이터를 제공하므로 일부 비상장사는 조회가 되지 않을 수 있습니다.

### 5.3 공시뷰어 연결 (User Experience)
사용자 UI에서 특정 데이터의 근거를 보여주기 위해 DART 공시 뷰어로 직접 연결하고 싶다면, 응답 데이터의 `rcept_no`를 활용하십시오.
- **PC용 주소**: `https://dart.fss.or.kr/dsaf001/main.do?rcpNo={rcept_no}`

### 5.4 API 호출 조합 방법
보통 서비스 개발 시 다음과 같은 순서로 데이터를 수집합니다.
1. `corp_code`를 검색하거나 캐시된 데이터에서 가져옵니다.
2. 해당 `corp_code`와 원하는 `bsns_year`, `reprt_code`를 조합하여 이 API를 호출합니다.
3. `fs_div`의 값이 `CFS`(연결)인지 `OFS`(개별)인지 확인하여 분석 목적에 맞는 데이터를 필터링합니다. 일반적으로 기업 가치 분석 시에는 `CFS` 데이터를 우선시합니다.

### 5.5 주의할 점: 단위(Unit)
금액 필드(`thstrm_amount` 등)는 문자열 타입으로 반환되지만, 내부에 숫자가 들어있습니다. 파싱 시 컴마(,)가 포함되어 있을 수 있으므로 전처리 후 정수/실수형으로 변환해야 합니다. 또한 기본 단위는 **원**입니다.
