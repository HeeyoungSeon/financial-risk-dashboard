# [OpenDART] 다중회사 주요계정 API 개발 가이드

이 문서는 OpenDART에서 제공하는 **다중회사 주요계정** API에 대한 상세 개발 가이드입니다. 이 API는 여러 회사(최대 100개)의 주요 재무 계정 정보를 한 번의 호출로 비교 분석하고자 할 때 매우 효율적입니다.

## 1. 기본 정보

| 항목 | 상세 내용 |
| :--- | :--- |
| **메서드** | GET |
| **요청 URL** | `https://opendart.fss.or.kr/api/fnlttMultiAcnt.json` (또는 `.xml`) |
| **인코딩** | UTF-8 |
| **출력 포맷** | JSON, XML |

---

## 2. 요청 인자 (Parameters)

| 요청키 | 명칭 | 타입 | 필수여부 | 값 설명 |
| :--- | :--- | :--- | :---: | :--- |
| `crtfc_key` | API 인증키 | STRING(40) | Y | 사용자가 발급받은 40자리의 인증키 |
| `corp_code` | 고유번호 | STRING | Y | **쉼표(,)로 구분된** 고유번호 리스트 (최대 100건 지원)<br>예: `00126380,00164779,00366997` |
| `bsns_year` | 사업연도 | STRING(4) | Y | 사업연도 (4자리, 2015년 이후 정보 제공) |
| `reprt_code` | 보고서 코드 | STRING(5) | Y | 1분기: `11013`, 반기: `11012`, 3분기: `11014`, 사업보고서: `11011` |

---

## 3. 응답 결과 (Response Fields)

결과는 `list` 내에 각 회사별, 각 계정별로 반복되어 나타납니다.

| 응답키 | 명칭 | 출력 설명 |
| :--- | :--- | :--- |
| `status` | 에러 코드 | 정상 시 `000` |
| `message` | 안내 메시지 | 에러 등 상세 내용 |
| `list` | 데이터 배열 | |
| `rcept_no` | 접수번호 | 해당 공시 접수번호 (14자리) |
| `bsns_year` | 사업 연도 | 요청한 사업 연도 |
| `stock_code` | 종목 코드 | 상장회사의 종목코드(6자리) |
| `reprt_code` | 보고서 코드 | 1분기, 반기, 3분기, 사업보고서 등 |
| `account_nm` | 계정명 | 자본총계, 자산총계, 부채총계, 매출액, 영업이익, 당기순이익 등 |
| `fs_div` | 개별/연결 구분 | `OFS`: 재무제표, `CFS`: 연결재무제표 |
| `fs_nm` | 개별/연결명 | "연결재무제표" 또는 "재무제표" |
| `sj_div` | 재무제표 구분 | `BS`: 재무상태표, `IS`: 손익계산서 |
| `sj_nm` | 재무제표명 | "재무상태표" 또는 "손익계산서" |
| `thstrm_nm` | 당기명 | 당기 분류명 |
| `thstrm_amount` | 당기금액 | 해당 계정의 금액 (단위: 원) |
| `thstrm_add_amount` | 당기누적금액 | 분기/반기 보고서의 경우 누적치 (단위: 원) |
| `frmtrm_nm` | 전기명 | 전기 분류명 |
| `frmtrm_amount` | 전기금액 | 전기 전체 금액 (단위: 원) |
| `frmtrm_add_amount` | 전기누적금액 | 전기 누적치 (단위: 원) |
| `bfefrmtrm_nm` | 전전기명 | 전전기 분류명 |
| `bfefrmtrm_amount` | 전전기금액 | 전전기 전체 금액 (단위: 원) |
| `ord` | 계정과목 정렬순서 | 고정된 출력 순서 |

---

## 4. 메시지 설명 (Status Codes)

- `000`: 정상
- `021`: 조회 가능한 회사 개수 초과 (최대 100건까지만 쉼표로 연결 가능)
- 기타 코드는 `api_corp_code.md` 참고

---

## 5. 개발 참고 사항 및 활용 팁

### 5.1 대량 조회의 이점
동종 업계 내 경쟁사들을 비교하거나 특정 지수(Index)에 포함된 기업들의 성과를 일괄 파싱할 때 매우 유용합니다. 개별 조회를 100번 할 필요 없이 하나의 요청으로 처리하므로 **네트워크 오버헤드와 API 쿼터 소모를 획기적으로 줄여줍니다.**

### 5.2 데이터 정렬 및 그룹화 (Post-processing)
응답으로 오는 `list` 배열은 요청한 `corp_code`의 순서대로 올 수도 있지만, 데이터 가공 과정에 따라 섞여 있을 수 있습니다. 따라서 개발 시에는 파이썬의 `pandas` 등을 활용하여 `stock_code`나 `corp_code`를 기준으로 **데이터프레임을 그룹화(Group by)** 하거나 피벗 테이블을 생성하여 시각화하는 것이 좋습니다.

### 5.3 누적 금액(`add_amount`)의 활용
이 API는 당기 금액뿐만 아니라 누적 금액(`thstrm_add_amount`)을 제공합니다. 특히 3분기 보고서의 경우, 3분기 단독 실적뿐만 아니라 1~3분기 전체 누적 실적을 비교할 수 있어 연간 실적 예측에 핵심적인 데이터가 됩니다.

### 5.4 비교 분석 예시 (가상 시나리오)
- **부채비율 계산**: (자본총계 / 부채총계) * 100 공식을 사용하여 여러 기업의 재무 건전성 비교 차트를 생성.
- **성장성 분석**: 전기 매출액 대 당기 매출액의 증감률을 계산하여 업종 내 성장주 발굴.
- **수익성 분석**: 매출액 대비 영업이익(영업이익률) 리스트를 정렬하여 효율성 순위 산출.

### 5.5 주의사항 및 제한
- 모든 기업이 모든 계정을 제공하지는 않습니다. 일부 정보가 누락된 경우 `thstrm_amount` 등이 비어있을 수 있으므로 예외 처리가 필요합니다.
- 상장사 간에도 회계 처리 방식에 따라 계정명(`account_nm`)이 미세하게 다를 수 있으나, 표준화된 계정은 이 API를 통해 일정 수준 보정되어 제공됩니다.
- 쉼표로 연결할 때 공백이 포함되지 않도록 주의하십시오 (예: `code1,code2` 권장).
